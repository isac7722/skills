# Restate — 상세 사용 가이드

## 개요

`/restate`는 요구사항 엔지니어링(Requirement Engineering) 기법을 활용하여, 사용자의 요청에 숨겨진 가정과 모호한 부분을 사전에 파악하는 스킬입니다.

핵심 원칙: **요청을 절대 실행하지 않고, 분석만 수행합니다.**

---

## 동작 방식

### 입력

`/restate` 명령 직전의 사용자 요청을 분석 대상으로 삼습니다.

- 직전 메시지뿐 아니라, 최근 5턴의 대화 이력을 참조하여 맥락을 파악합니다.
- "이 파일", "그 모듈", "그것" 같은 대명사는 대화 맥락에서 자동으로 해석됩니다.

### 출력 구조

Claude는 4가지 섹션으로 구성된 분석 결과를 제공합니다:

#### 1. 핵심 요청 (Core Request)

요청을 한 문장으로 요약합니다.

> 예: "인증 모듈에 더 나은 에러 핸들링을 추가해달라는 요청입니다."

#### 2. 이해한 내용 (My Understanding)

| 항목 | 설명 |
|------|------|
| **목표 (Goal)** | 달성하려는 것이 무엇인지 |
| **범위 (Scope)** | 어디에, 무엇에 적용되는지 |
| **제약사항 (Constraints)** | 고려해야 할 특수 조건 |
| **가정 (Assumptions)** | Claude가 추론한 암묵적 가정 |

#### 3. 불명확한 부분 (Unclear Aspects)

모호한 점을 구체적인 질문 형태로 나열합니다.

> 예:
> - "에러 핸들링"이란 try-catch 추가를 의미하나요, 아니면 사용자 친화적 에러 메시지를 의미하나요?
> - 기존 에러 핸들링 로직은 유지하나요, 교체하나요?

#### 4. 확인 (Confirmation)

"제가 이해한 내용이 맞나요?" 라고 물어봅니다.

---

## 사용 시나리오

### 시나리오 1: 모호한 요청 명확화

```
사용자: API 성능을 개선해줘
사용자: /restate
```

Claude가 "성능 개선"이 응답 시간 단축인지, 처리량 증가인지, 메모리 최적화인지 등을 질문으로 구분해줍니다.

### 시나리오 2: 대화 맥락 기반 해석

```
사용자: user.py 파일을 살펴보자
사용자: 여기에 캐싱을 추가해줘
사용자: /restate
```

Claude가 "여기"가 `user.py`를 가리킨다는 것을 대화 맥락에서 파악하고, 캐싱의 대상과 전략에 대해 구체적으로 질문합니다.

### 시나리오 3: 복잡한 리팩토링 전 확인

```
사용자: 이 프로젝트의 인증 시스템을 JWT 기반으로 전환해줘
사용자: /restate
```

Claude가 기존 세션 방식 폐기 여부, 토큰 갱신 전략, 마이그레이션 범위 등 숨겨진 결정 사항들을 표면화합니다.

---

## 응답 언어

Claude는 **현재 대화에서 사용 중인 언어**로 응답합니다. 한국어로 대화 중이면 한국어로, 영어로 대화 중이면 영어로 재진술합니다.

---

## 특수 케이스 처리

| 상황 | Claude의 반응 |
|------|--------------|
| 이전 요청이 없는 경우 | "재진술할 요청이 없습니다. 먼저 요청을 제공해 주세요." |
| 요청이 이미 명확한 경우 | 재진술을 수행하되, "이 요청은 이미 매우 명확합니다"라고 언급 |
| 요청이 너무 모호한 경우 | "이 요청은 너무 간략합니다. 더 많은 세부사항을 제공해 주시겠어요?" |

---

## 제한 사항

`/restate` 실행 시 Claude는 다음을 **하지 않습니다**:

- 원래 요청을 실행
- 파일 읽기/수정
- 명령어 실행
- 코드 작성

오직 **분석과 질문**만 수행합니다.

---

## 성공/실패 판단 기준

### 성공

- 사용자가 "맞아", "정확해", "그래" 등으로 확인
- 최소 1개의 숨겨진 가정을 표면화
- 모호한 부분이 구체적 질문으로 전환됨

### 실패

- 사용자가 "그게 아니야"라고 응답
- 원래 요청에 없는 개념을 도입
- 확인 없이 작업을 시작

---

## 팁

- 복잡한 작업을 시작하기 전에 습관적으로 `/restate`를 사용하면 재작업을 줄일 수 있습니다.
- `/restate` 후 수정 사항을 반영하고, 필요하면 다시 `/restate`를 사용하여 이해도를 점진적으로 높일 수 있습니다.
