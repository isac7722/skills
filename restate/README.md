# Restate

사용자의 요청을 실행하기 전에, 요청을 정확히 이해했는지 확인하는 스킬입니다.

## 한 줄 요약

`/restate`는 요청을 **실행하지 않고**, 대신 요청을 재진술하여 상호 이해를 확인합니다.

## 사용법

1. Claude에게 작업을 요청합니다.
2. `/restate`를 입력합니다.
3. Claude가 요청을 4가지 관점으로 분석하여 보여줍니다.
4. 맞으면 확인, 틀리면 수정합니다.

## 사용 예시

```
사용자: 인증 모듈의 에러 핸들링을 개선해줘
사용자: /restate
```

Claude가 다음을 분석하여 응답합니다:
- **핵심 요청**: 요청의 한 줄 요약
- **이해한 내용**: 목표, 범위, 제약사항, 가정
- **불명확한 부분**: 구체적 질문 목록
- **확인 요청**: 이해가 맞는지 확인

## 언제 사용하면 좋을까?

- 복잡하거나 모호한 요청을 할 때
- 대규모 리팩토링이나 변경 전 방향을 확인하고 싶을 때
- 숨겨진 가정이나 빠진 조건을 사전에 발견하고 싶을 때
