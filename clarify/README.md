# Clarify

모호한 요구사항을 반복적인 질문을 통해 명확하고 실행 가능한 스펙으로 변환하는 스킬입니다.

## 언제 사용하나요?

- 요구사항이 모호하거나 불완전할 때
- 구현 전에 정확한 의도를 파악하고 싶을 때
- 여러 해석이 가능한 요청을 정리하고 싶을 때

## 사용 방법

Claude Code에서 아래 키워드로 트리거합니다:

```
/clarify
```

또는 자연어로 요청:

```
"요구사항 명확하게 해줘"
"이거 좀 정리해줘"
"clarify requirements"
```

## 간단한 예시

**입력**: "로그인 기능 추가해줘"

**clarify가 하는 일**:
1. 원본 요구사항 기록
2. 모호한 부분을 하나씩 질문 (인증 방식? 세션 시간? 비밀번호 규칙?)
3. Before/After 비교표 제공
4. 필요시 파일로 저장

**결과**: "bcrypt 암호화, 24시간 세션, 비밀번호 8자 이상의 이메일/비밀번호 로그인 및 셀프 회원가입 기능 구현"
